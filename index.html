M1K3 Architecture

A lightweight, local AI CLI powered by small language models, featuring voice synthesis and system-aware dynamic context.

Core Components

🧠AI Engine

Handles model loading, inference, and context management. Swappable between a real GGUF-based engine and a mock engine.

Key Files

ai_inference.py simple_ai_engine.py

🎮CLI Interface

The main user interface, managing state, animations, user commands, and orchestrating the other components.

Key Files

cli.py cli_animations.py

🔊Voice Engine

Provides text-to-speech synthesis with a layered approach, preferring advanced models and falling back to system TTS.

Key Files

enhanced_voice_engine.py zen_voice_engine.py simple_voice_engine.py

📊System Monitor

Collects privacy-safe system metrics like CPU, memory, and battery to provide dynamic, context-aware greetings.

Key Files

system_metrics.py

⬇️Model Manager

Handles downloading and verifying GGUF model files from HuggingFace, with device analysis for recommendations.

Key Files

download_model.py

🛠️Utilities

Supporting scripts for development, including performance benchmarking and demonstrations.

Key Files

benchmark.py Makefile

System Flow

  • 1

    Initialization

    The CLI starts, collects system metrics, and determines which AI engine to use (real or mock).

  • 2

    Model Loading

    The AI engine checks for a local model. If not found, the Model Manager downloads it before loading into memory.

  • 3

    Dynamic Greeting

    The System Monitor generates a personalized greeting based on time of day, battery status, and CPU load.

  • 4

    Interactive Loop

    The CLI waits for user input, processes commands, and passes prompts to the AI engine for a streaming response.

  • 5

    Voice Synthesis

    The generated text response is sent to the Voice Engine, which synthesizes and plays the audio in the background.
